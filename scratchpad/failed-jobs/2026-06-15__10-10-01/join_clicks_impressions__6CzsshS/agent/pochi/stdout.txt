## What it does:

1. **Reads** `impressions.jsonl` and `clicks.jsonl` via `FileSource`
2. **Parses** each line from JSON string to Python dict
3. **Casts** `ad_id` in clicks from string to int (handling the type mismatch)
4. **Keys** both streams by `str(ad_id)` (Bytewax requires string routing keys)
5. **Joins** the keyed streams using `op.join` (inner join, complete emit mode)
6. **Formats** the result as `{"ad_id": N, "user_id": "...", "click_time": "..."}` JSON strings
7. **Outputs** to `joined.jsonl` via `FileSink`

## Key design decisions for recovery compatibility:

- All mapper/key functions are **module-level named functions** (not lambdas), ensuring they are picklable
- The output of `format_joined` is a `(key, value)` 2-tuple as required by `op.output`
- No generators, file handles, or other unpicklable objects are stored in state
