## Implementation Summary

**Dataflow pipeline:**
1. `op.input` → reads JSON lines from `/home/user/bytewax_project/input.jsonl` via `FileSource`
2. `op.map("deserialize")` → parses each line into a Python dict
3. `op.key_on("key_on_user")` → keys each event by `str(user_id)` for stateful routing
4. `op.stateful_map("token_bucket")` → applies per-user token bucket rate limiting
5. `op.filter("filter_allowed")` → drops throttled events (where value is `None`)
6. `op.map("serialize")` → serializes allowed events back to JSON strings (keeping keys for routing)
7. `op.output` → writes to `/home/user/bytewax_project/output.jsonl` via `FileSink`

**Token bucket logic (`token_bucket_mapper`):**
- State is a picklable tuple of `(current_tokens: float, last_timestamp: float)`
- Max capacity = 5.0 tokens, initial tokens = 5.0, refill rate = 1.0 token/sec
- On each event: calculate elapsed time since last event, refill tokens (capped at 5.0)
- If tokens ≥ 1.0: subtract 1.0 and allow the event
- If tokens < 1.0: do not subtract and drop the event (emit `None`)
- Last timestamp is always updated to the current event's timestamp

**Picklability:** The state is a plain `tuple[float, float]` which is fully picklable. All mapper functions are top-level module functions, also picklable by reference. This supports Bytewax's SQLite recovery mechanism.

**Verified behavior:**
- 10 burst events at the same timestamp → first 5 allowed, last 5 dropped
- Events spaced out over time → all allowed (refill keeps up with demand)
- State snapshot and resume work correctly via pickle round-trip
