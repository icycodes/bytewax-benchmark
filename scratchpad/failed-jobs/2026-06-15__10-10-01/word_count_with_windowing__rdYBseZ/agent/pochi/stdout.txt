## Implementation Summary

**Usage:** `python dataflow.py <input_file> <output_file>`

**Architecture:**

1. **`JSONLSource`** — A `DynamicSource` that reads JSON Lines input, parsing each line into a `dict` with `"time"` and `"word"` keys.

2. **`EventClock`** — Uses a `ts_getter` function that parses ISO8601 timestamps (both `Z` suffix and `+00:00` offset formats) into UTC `datetime` objects via `parse_iso8601_to_utc()`.

3. **`TumblingWindower`** — 1-hour tumbling windows aligned to `2023-01-01T00:00:00Z`.

4. **`count_window`** — Groups events by word (via the `key` parameter extracting `event["word"]`), counts occurrences per window using event time.

5. **Output formatting** — The `windowed.down` stream yields `(word, (window_id_int, count))` tuples. The windower logic's `_metadata_for()` method resolves the integer window ID to a `WindowMetadata` with `open_time`, which is formatted as an ISO8601 string.

6. **`JSONLSink`** — A `DynamicSink` that writes each result dict as a JSON line to the output file.

**Verification results** with 8 test events across 3 windows matched expected counts exactly:
- Window `2023-01-01T00:00:00+00:00`: hello=2, world=1
- Window `2023-01-01T01:00:00+00:00`: hello=1, world=1
- Window `2023-01-01T02:00:00+00:00`: hello=1, world=2
