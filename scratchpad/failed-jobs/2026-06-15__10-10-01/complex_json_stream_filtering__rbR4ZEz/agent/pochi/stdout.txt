**How it works:**

1. **Input**: Reads `input.jsonl` via `FileSource`
2. **Parse & Validate**: `filter_map` with `parse_json` — parses JSON, returns `None` for invalid JSON or records missing `type`, `device_id`, or `payload` fields
3. **Key by device**: `key_on` on `device_id` so stateful operations are partitioned per device
4. **Stateful processing**: `stateful_map` maintains threshold state per device (default `100.0`):
   - `config` events update the threshold from `payload.threshold`
   - `metric` events compare `payload.temperature` against the current threshold, emitting a JSON alert string if exceeded
5. **Filter**: `filter_map` drops `None` values (non-alert events)
6. **Output**: Writes alert JSON strings to `output-${ZEALT_RUN_ID}.jsonl` via `FileSink`
