**How it works:**

1. **Three `FileSource` inputs** read `users.jsonl`, `emails.jsonl`, and `activities.jsonl` line-by-line.
2. **Parsing + keying**: Each line is parsed as JSON, the user ID field is extracted and normalized to a string key (handling both `int` and `str` IDs in the source data), and the relevant value is emitted as a `(key, value)` tuple.
3. **`op.join` with `emit_mode="complete"`**: Performs an inner join across all three streams — only emits when all three sides have a value for a given key.
4. **Formatting**: The joined tuple `(key, (name, email, last_login))` is formatted into a JSON string with keys `id`, `name`, `email`, `last_login`.
5. **`StdOutSink`** writes the JSON strings to stdout.
