## How it works

1. **Input** — `FileSource` reads the JSONL file line by line.
2. **Parse** — each line is decoded into a `(word, datetime)` tuple; the ISO 8601 timestamp is parsed into a UTC-aware `datetime`.
3. **Event clock** — `EventClock` extracts the timestamp from each tuple. `wait_for_system_duration=timedelta(0)` advances the watermark immediately, which is correct for bounded/batch input.
4. **Windowing** — `TumblingWindower(length=1h, align_to=2023-01-01T00:00:00Z)` places every event into exactly one 1-hour bucket.
5. **Count** — `count_window` keys events by their word and counts occurrences per window. It emits `(word, (window_id_int, count))` tuples.
6. **Format** — the integer `window_id` is converted to the window's UTC open time via `align_to + window_id * 1h`, then serialised to a JSON string.
7. **Output** — a custom `JsonlFileSink` (a `DynamicSink`) writes one JSON object per line.
