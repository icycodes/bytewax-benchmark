## What was built

### `input.jsonl`
13 lines of mixed IoT events: valid config/metric events, invalid JSON, and a record missing required fields.

### `pipeline.py` — dataflow stages

```
FileSource(input.jsonl)
  → filter_map("parse_and_validate")   # parse JSON; drop invalid/missing-field lines
  → stateful_map("process_event")      # key=device_id; track threshold, emit alert dicts
  → filter_map("serialize_alert")      # drop None (non-alerts); serialise to JSON string
  → FileSink(output-${run_id}.jsonl)
```

**`parse_and_validate`** — returns `(device_id, event)` or `None`:
- Catches `json.JSONDecodeError` silently
- Rejects any record missing `type`, `device_id`, or `payload`
- Rejects non-string `device_id`

**`process_event`** — stateful mapper `(state, event) → (new_state, alert | None)`:
- State is the current threshold, initialised to `100.0` on first sight of a device
- `"config"` events update the threshold in state, emit `None`
- `"metric"` events compare temperature strictly against threshold; emit an alert dict only when `temp > threshold`

**`serialize_alert`** — keeps the key intact as `(device_id, json_string)` so `FileSink` (which requires a keyed 2-tuple) receives the correct shape.
