## connectors.py – key design decisions

### `CustomFileSource` (`FixedPartitionedSource`)
- `list_parts()` returns sorted `.txt` filenames found in `input_dir`.
- `build_part()` creates a `_FileSourcePartition` that opens the file and skips the first `resume_state` lines (fast-forward on recovery).
- `next_batch()` returns one `(filename, line)` tuple per call; raises `StopIteration` at EOF.
- `snapshot()` returns `_next_line` — the 0-based index of the *next* line to read.

### `CustomFileSink` (`FixedPartitionedSink`)
- `list_parts()` mirrors the source so partition indices are globally consistent.
- `part_fn(key)` returns `sorted_parts.index(key)`, ensuring each `(filename, line)` item routes to the partition whose name matches `filename` exactly — regardless of worker count.
- `build_part()` creates a `_FileSinkPartition` which **truncates the output file to `resume_state` lines** before opening it for append, removing any partial data from a crashed run.
- `snapshot()` returns the total number of lines written so far.
