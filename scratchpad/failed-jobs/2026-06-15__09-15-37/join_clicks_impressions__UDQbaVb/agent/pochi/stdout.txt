**Key design decisions:**

1. **Named module-level functions** — all mappers (`parse_json`, `impression_key`, `click_key`, `format_joined`) are defined as regular named functions at module level. Lambdas are *not* picklable, which would crash SQLite recovery snapshots.

2. **Type mismatch resolution** — `impression_key` casts `ad_id` (int) to `str` via `str(record["ad_id"])`. `click_key` normalises the already-string `ad_id` the same way. This makes both routing keys identical (`"1"`, `"2"`, `"3"`), enabling the join to match records correctly.

3. **`map_value` instead of `map` for formatting** — `FileSink` is a `FixedPartitionedSink` and requires a *keyed* `(str, str)` stream. Using `op.map_value` transforms only the value side of the joined keyed stream into a JSON string, preserving the key so routing works correctly.

4. **`emit_mode="complete"`** — the inner join only emits when *both* sides have a record for a given key, matching the expected inner-join semantics (ad ID 2, which has an impression but no click, is correctly omitted).
