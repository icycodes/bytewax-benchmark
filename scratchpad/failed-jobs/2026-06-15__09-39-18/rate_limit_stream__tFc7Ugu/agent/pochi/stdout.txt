**Key implementation details:**

- **Token Bucket State**: Uses `op.stateful_map` with state as a picklable tuple `(current_tokens, last_timestamp)`. Initial state is `None` (first event per user starts with `MAX_TOKENS = 5.0`).

- **Rate Limiting Logic**:
  - On first event: tokens = 5.0 (no elapsed time refill)
  - On subsequent events: `elapsed = event_timestamp - last_timestamp`, refill `min(current_tokens + elapsed * 1.0, 5.0)`
  - If tokens ≥ 1.0: subtract 1.0, allow the event
  - If tokens < 1.0: don't subtract, drop the event
  - Last timestamp is always updated to the current event's timestamp (whether allowed or dropped)

- **Pipeline Flow**:
  1. `FileSource` reads JSON lines from `input.jsonl`
  2. `json.loads` parses each line
  3. `key_on` keys by `user_id` (cast to string)
  4. `stateful_map` applies the token bucket logic
  5. `filter_value` drops `None` (throttled) events
  6. `map_value` serializes allowed events to JSON strings
  7. `FileSink` writes to `output.jsonl`

- **Fault Tolerance**: State tuples `(float, float)` are fully picklable, supporting Bytewax's SQLite recovery mechanism.

Run with: `python -m bytewax.run dataflow:flow`
