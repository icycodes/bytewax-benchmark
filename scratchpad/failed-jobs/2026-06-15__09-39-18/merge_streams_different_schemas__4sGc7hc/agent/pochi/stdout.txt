**How it works:**

1. **Three input streams** are created using `FileSource` to read `users.jsonl`, `emails.jsonl`, and `activities.jsonl`
2. Each stream is **parsed** from JSON (`json.loads`), then **keyed** on user ID (normalized to string via `str()` to handle mixed int/str IDs across files), and **mapped** to extract just the relevant value field
3. The three streams are **inner-joined** using `ops.join()` (default `emit_mode="complete"` + `insert_mode="last"`), which only emits when all three sides have data for a key
4. The joined result is **formatted** into a JSON string with the required keys (`id`, `name`, `email`, `last_login`) and output to stdout via `StdOutSink`
