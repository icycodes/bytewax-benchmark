Created a Bytewax stateful moving average pipeline at `/home/user/bytewax-task/pipeline.py`.

**Pipeline architecture:**
1. **Input** (`CsvSource`): Custom `FixedPartitionedSource` that reads `input.csv` line by line, parsing each line into `(sensor_id, temperature)` tuples.
2. **Key partitioning** (`op.key_on`): Routes data by `sensor_id` for per-sensor stateful processing.
3. **Value extraction** (`op.map_value`): Extracts the temperature float from the `(sensor_id, temperature)` tuple.
4. **Stateful moving average** (`op.stateful_map`): The `moving_avg` function maintains a list of up to 3 recent temperatures per sensor. It returns a **new state list** (never mutates in-place) to ensure recovery snapshot compatibility. The average is rounded to 2 decimal places.
5. **Formatting** (`op.map`): Formats the `(sensor_id, avg)` tuple as `sensor_id,avg` string.
6. **Output** (`CsvSink`): Custom `DynamicSink` that writes each line to `output.csv`.

**Key design decisions:**
- State is immutable — `state = state + [value]` creates a new list rather than mutating, satisfying Bytewax's recovery snapshot requirement.
- Keys are strings (sensor IDs like `"s1"`, `"s2"`) as required.
- When a sensor has fewer than 3 readings, the average of all available readings is computed.
- Output format: `sensor_id,moving_average` (e.g., `s1,20.50`) with exactly one line per input line.

**Run command:** `python -m bytewax.run pipeline:flow` (from `/home/user/bytewax-task/`)
