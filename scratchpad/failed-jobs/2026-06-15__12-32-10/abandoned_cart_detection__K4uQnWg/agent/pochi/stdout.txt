## How it works

1. **Input**: Reads JSON-encoded events line-by-line from the file specified by `INPUT_FILE` env var via `FileSource`.

2. **Parsing & Keying**: Each line is parsed as JSON, then keyed by `user_id` using `key_on`.

3. **Stateful Cart Logic** (`CartLogic`):
   - **`on_item("add_to_cart")`**: Appends the item to the cart. On the first item, sets an expiration time (`now + CART_TIMEOUT_SECONDS`).
   - **`on_item("checkout")`**: Clears the cart and discards the logic (no abandonment emitted).
   - **`on_notify()`**: Called when the timeout expires — emits `{"abandoned_items": [...]}` and discards the logic.
   - **`on_eof()`**: Called at stream end — emits any remaining non-empty carts as abandoned and discards.
   - **`notify_at()`**: Returns the expiration `datetime` (or `None` if no timeout is active).
   - **`snapshot()`**: Returns recoverable state for fault tolerance.

4. **Output**: Results are formatted as JSON lines and written to `OUTPUT_FILE` via `FileSink`.

## Key design decisions

- **Timeout starts on first item**: Per the spec, `CART_TIMEOUT_SECONDS` countdown begins when the first item is added to a cart, not on subsequent additions.
- **Checkout discards state**: A `checkout` event immediately clears the cart and discards the per-key logic, preventing any abandonment emission.
- **EOF flushes remaining carts**: Any non-empty carts still in memory when the stream ends are emitted as abandoned, regardless of whether the timeout has elapsed.
- **`CART_TIMEOUT_SECONDS`** defaults to 900 (15 minutes) if not set in the environment.
