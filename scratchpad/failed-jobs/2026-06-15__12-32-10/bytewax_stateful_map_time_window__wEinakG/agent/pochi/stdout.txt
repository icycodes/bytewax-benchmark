**Pipeline architecture:**

1. **Input** — Reads JSON lines from `events.json` via `FileSource`
2. **Parse** — `op.map` parses each line into a dict
3. **Key** — `op.key_on` keys the stream by `user_id`
4. **Deduplicate** — `op.stateful_map` maintains per-user state (`event_id → datetime`). For each event:
   - Prunes state entries older than 10 seconds relative to the current event's timestamp
   - If the `event_id` exists in the pruned state, it's a duplicate → emit `None`
   - Otherwise, add to state and emit the event
5. **Filter** — `op.filter_value` removes `None` values (duplicates)
6. **Format** — `op.map_value` serializes each event dict back to a JSON string
7. **Output** — `op.output` writes to `output.json` via `FileSink`
