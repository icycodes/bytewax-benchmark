### Architecture walkthrough

#### `AsyncFileSource` (DynamicSource)
- `build(step_id, worker_index, worker_count)` — lists and sorts all `.txt` files in the given directory, then slices `files[worker_index::worker_count]` to give each worker a disjoint, non-overlapping subset.

#### `_AsyncFilePartition` (StatelessSourcePartition)
- **`__init__`** — creates a `queue.Queue` and launches a **daemon `threading.Thread`** that owns its own `asyncio` event loop (via `asyncio.new_event_loop()`).
- **`_run_event_loop` / `_read_files_async`** — the background coroutine uses `loop.run_in_executor(None, ...)` to offload blocking file I/O onto the default thread-pool executor, simulating a real async network client. After all files are read, a `None` sentinel is placed in the queue.
- **`next_batch`** — fully synchronous. It does a short-timeout `queue.get(timeout=0.05)` to avoid busy-spinning. It drains all immediately-available items into a list and returns it. When the `None` sentinel is dequeued (and the queue is empty), it raises `StopIteration`, which Bytewax uses to signal end-of-input for this partition.

#### `StdOutSink` (DynamicSink)
- A minimal custom sink (Bytewax 0.21 has no built-in `StdOutSink`) that writes each item with `print(..., flush=True)`.

#### Dataflow

stream      = op.input("file_input", flow, AsyncFileSource("input_data"))

op.output("stdout", upper_stream, StdOutSink())
