## What was built

**`input.csv`** — sample sensor readings (6 rows, two sensors `s1` and `s2`).

**`pipeline.py`** — a Bytewax 0.21 dataflow named `flow` with these steps:

| Step | Operator | Purpose |
|---|---|---|
| 1 | `op.input` + `FileSource` | Read `input.csv` line-by-line |
| 2 | `op.map` | Parse `"sensor_id,temp"` → `(str, float)` keyed tuple |
| 3 | `op.stateful_map` | Per-sensor sliding window of last 3 readings → moving average |
| 4 | `op.map` | Format result as `"sensor_id,xx.xx"` |
| 5 | `op.output` + `FileSink` | Write `output.csv` |

**Key design decisions:**
- State is an **immutable tuple** — a new tuple is constructed on every call (`(state + (temp,))[-WINDOW:]`), never mutated in-place, ensuring correct recovery snapshots.
- Handles the < 3 readings edge case naturally: the slice `[-3:]` on a shorter tuple simply keeps all elements, so the average is computed over whatever is available.
