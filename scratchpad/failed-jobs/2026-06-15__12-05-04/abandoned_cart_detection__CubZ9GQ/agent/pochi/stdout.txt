## Design overview

### Source
`FileSource(INPUT_FILE)` reads the file line-by-line as raw strings.

### Parse step (`filter_map`)
`_parse_event` decodes each JSON line and returns a `(user_id, event)` keyed tuple, or `None` for blank/malformed lines (automatically filtered).

### Stateful operator (`op.stateful`)
`CartLogic` (subclass of `StatefulLogic`) manages per-user cart state:

| Method | Behaviour |
|---|---|
| `on_item` | `add_to_cart` → append item, start expiry clock on first add. `checkout` → clear state, **DISCARD** (no emit). |
| `on_notify` | Timeout elapsed → emit abandonment JSON, **DISCARD**. |
| `on_eof` | Stream ended with items still in cart → emit abandonment JSON, **DISCARD**. |
| `notify_at` | Returns the scheduled expiry `datetime` (UTC) so the runtime fires `on_notify` at the right time. |
| `snapshot` | Returns `(items_copy, expiry_iso_str)` for recovery. |

Because `user_id` is the stream key (injected by `op.stateful` wrapping), the emitted JSON carries a `null` placeholder that a downstream `op.map` step replaces with the real key.

### Sink
A custom `JsonLinesSink` (`DynamicSink`) truncates the output file at startup and appends one JSON line per abandoned cart.

### Verified scenarios
- **EOF flush** – non-checkout carts pending when input ends are emitted immediately.  
- **Checkout clears cart** – checked-out users produce no output.  
- **Timeout via `on_notify`** – carts with `CART_TIMEOUT_SECONDS=1` are emitted by the notification callback before EOF.
