## How it works

```
JSONL file
    │
    ▼ op.input  (FileSource)
raw_lines  ──► op.map  (json.loads)
                    │
                    ▼  events (dict stream)
         op.branch("route_error",  is_error)
         ├── trues  ──► op.map(json.dumps) ──► errors.jsonl
         └── falses
                │
         op.branch("route_metric", is_metric)
         ├── trues  ──► op.map(json.dumps) ──► metrics.jsonl
         └── falses
                │
         op.branch("route_log",    is_log)
         ├── trues  ──► op.map(json.dumps) ──► logs.jsonl
         └── falses ──► op.map(json.dumps) ──► dead_letter.jsonl
```

### Key design decisions

| Concern | Choice |
|---|---|
| **Routing mechanism** | Cascaded `op.branch()` operators — each peels off one category; the `falses` stream flows into the next branch |
| **Sink** | Custom `JsonlFileSink(DynamicSink)` + `_JsonlSinkPartition(StatelessSinkPartition)` — truncates on creation, appends per batch, compatible with multi-worker runs |
| **Dead letter** | Events with a missing or unrecognised `type` fall through all three branches naturally; JSON parse failures are wrapped in an error envelope and also land in `dead_letter.jsonl` |
| **File output location** | Same directory as `run.py`, determined at runtime via `Path(__file__).parent` |
