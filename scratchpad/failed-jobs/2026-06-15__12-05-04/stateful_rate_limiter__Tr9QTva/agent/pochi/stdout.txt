## Implementation Summary

### Architecture (Bytewax 0.21.1)

```
TestingSource(raw_lines)
    → op.map("parse_json")         # str → dict
    → op.key_on("key_by_user")     # dict → (user_id_str, dict)
    → op.stateful_map("token_bucket", token_bucket_mapper)
    → op.output("output", FileSink)
```

### Token Bucket Logic (`token_bucket_mapper`)

| Scenario | Behaviour |
|---|---|
| First event for a key | State initialised to `(MAX_TOKENS=10.0, event.timestamp)` |
| Subsequent events | Refill `elapsed × 2.0` tokens, cap at 10.0 |
| `cost ≥ 0`, bucket sufficient | Deduct cost → `allowed: true` |
| `cost ≥ 0`, bucket insufficient | Bucket unchanged → `allowed: false` |
| `cost == -1` (reset) | Restore to 10.0 tokens → `allowed: true` |

### Custom Sink (`FileSink / _FileSinkPartition`)

Extends `DynamicSink` / `StatelessSinkPartition` to write keyed `(user_id, record)` pairs as JSON lines to the output file.
