## What `flow.py` does

1. **Input** — `FileSource` reads `input.jsonl` line-by-line.
2. **Parse** — each raw JSON string is decoded into a `(user_id, event_dict)` keyed tuple so Bytewax can route it by user.
3. **Clock** — `EventClock` extracts the ISO-8601 `timestamp` field from each event (converted to an aware UTC `datetime`). `wait_for_system_duration=timedelta(seconds=0)` is used because this is a bounded/batch file source.
4. **Windowing** — `SessionWindower(gap=timedelta(seconds=5))` groups events per user: a new session begins whenever two consecutive events for the same user are more than 5 seconds apart.
5. **Collect** — `collect_window` gathers all events in each session into an ordered list.
6. **Format** — the `(user_id, (window_id, events))` tuple is mapped to `(user_id, json_string)` — the key-value pair required by `FileSink`.
7. **Output** — `FileSink` writes one JSON line per session to `output.jsonl`.
