### How it works

**Dataflow stages**

| Step | Operator | What it does |
|---|---|---|
| `read_events` | `op.input` + `FileSource` | Reads `events.json` line-by-line, yielding raw JSON strings |
| `key_by_user` | `op.key_on` | Extracts `user_id` as the string key → `Stream[(user_id, event_json)]` |
| `dedup` | `op.stateful_map` | Per-user deduplication with state pruning (see below) |
| `drop_dupes` | `op.filter_map` | Removes entries where the mapper emitted `None` (duplicates) |
| `write_output` | `op.output` + `FileSink` | Writes surviving `(key, value)` tuples to `output.json` |

**`dedup_mapper` logic** (called once per event, per user):
1. **Prune state** — remove every `event_id` whose recorded timestamp is more than 10 seconds older than the current event's timestamp. This prevents unbounded state growth.
2. **Duplicate check** — if `event_id` is still in state after pruning → duplicate → return `(state, None)`.
3. **New event** — record `event_id → current_ts` in state and return `(state, event_json)`.
