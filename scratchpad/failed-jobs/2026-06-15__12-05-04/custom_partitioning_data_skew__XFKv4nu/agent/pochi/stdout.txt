## `dataflow.py` — Key Design

### `RoundRobinFilePartition` (subclasses `StatefulSourcePartition`)
- Opened on construction with `partition_id` (0, 1, or 2) and an optional `resume_state` (the next global line index for fault-tolerance/recovery).
- **`next_batch()`** advances through the file one line at a time. It only returns the line when `global_line_index % 3 == partition_id`, skipping all other lines. Raises `StopIteration` at EOF.
- **`snapshot()`** returns `self._next_line_idx` — the global index of the *next* line to emit — so Bytewax can resume correctly after a failure.
- **`close()`** closes the file handle.

### `RoundRobinJsonlSource` (subclasses `FixedPartitionedSource`)
- **`list_parts()`** returns `["0", "1", "2"]` — the three partition keys.
- **`build_part()`** constructs a `RoundRobinFilePartition` for the given `for_part` key, forwarding any `resume_state`.

### Dataflow pipeline
```
input("read_jsonl")   →  RoundRobinJsonlSource  (3 partitions, round-robin)
map("key_lines")      →  wrap each str line as ("out", line) tuple
output("write_jsonl") →  FileSink(output.jsonl)
```
`FileSink` is a `FixedPartitionedSink` and requires `(key, value)` tuples for routing — the fixed key `"out"` funnels all output into the single file partition.
