### Architecture and Design

The implementation uses **Bytewax 0.21.1**'s low-level `StatefulLogic` class to manage time-based expiration and cart state.

1. **Input Step (`FileSource`)**:
   Reads JSON-encoded events line-by-line from the file specified by the `INPUT_FILE` environment variable.
2. **Parsing & Filtering (`filter_map`)**:
   Parses each line as JSON, handling empty lines or invalid JSON gracefully.
3. **Keying (`key_on`)**:
   Keys the event stream by `user_id` so that state is maintained on a per-user basis.
4. **Stateful Operator (`stateful`)**:
   Uses the custom `CartLogic(StatefulLogic)` to manage the state of each user's cart:
   - **`on_item`**: When an item is added, it checks if it is the first item in the cart. If so, it records the current timestamp (`first_added_at`). On `checkout`, it immediately discards the state and clears the cart (no abandonment event emitted).
   - **`notify_at`**: Schedules the timeout to trigger at exactly `first_added_at + CART_TIMEOUT_SECONDS`.
   - **`on_notify`**: Triggered when the timeout expires. It emits the abandonment event and discards the cart state.
   - **`on_eof`**: Triggered when the stream ends. It flushes any remaining non-empty carts as abandoned.
   - **`snapshot`**: Safely returns an immutable copy of the state for Bytewax's recovery system.
5. **Formatting (`map`)**:
   Formats the emitted abandonment events into `(user_id, json_string)` tuples suitable for the file sink.
6. **Output Step (`FileSink`)**:
   Writes exactly one JSON object per line to the file specified by the `OUTPUT_FILE` environment variable.
