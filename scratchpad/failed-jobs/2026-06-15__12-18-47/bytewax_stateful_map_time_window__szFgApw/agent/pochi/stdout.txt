Here is a summary of the implementation:
1. **Dataflow Setup**: We read the input events from `events.json` using `FileSource` and parse each JSON line into a Python dictionary.
2. **Keying**: We use `op.key_on` to key the stream by the `user_id` field (which is a string, conforming to Bytewax's requirements).
3. **Stateful Deduplication**: We use `op.stateful_map` to maintain a state dictionary per `user_id`. The state maps seen `event_id`s to their parsed `datetime` timestamps.
    - **State Pruning**: Before checking for duplicates, we iterate over the state and remove any `event_id`s whose timestamp is strictly older than 10 seconds compared to the current event's timestamp. This ensures memory does not grow unbounded.
    - **Deduplication Logic**: We check if the current `event_id` exists in the pruned state. If it does (meaning it was seen `<= 10` seconds ago), we consider it a duplicate and emit `None`. If it's a new event, we add it to the state with its timestamp and emit the event.
4. **Formatting and Output**: We use `op.filter_map` to drop the duplicate `None` values and format the valid events back into JSON strings, paired with their key as required by `FileSink`. Finally, `op.output` writes the valid, deduplicated JSON lines to `output.json`.
