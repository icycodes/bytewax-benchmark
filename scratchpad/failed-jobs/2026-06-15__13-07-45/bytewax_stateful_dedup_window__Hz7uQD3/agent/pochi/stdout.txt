**How it works:**

1. **Input**: Reads JSON lines from `input.jsonl` via `FileSource`.
2. **Parse**: Each line is parsed into a dict with the ISO 8601 timestamp converted to a `datetime` object.
3. **Key by user_id**: Events are grouped by `user_id` so state is maintained per-user.
4. **Stateful dedup**: A `DedupLogic` class (extending `StatefulLogic`) maintains a dict mapping `event_id → last_emitted_timestamp`. On each event:
   - Stale entries (older than 10s from the current event's timestamp) are cleaned up to prevent memory leaks.
   - If the `event_id` hasn't been seen before, or if more than 10 seconds have elapsed since the last emission, the event is emitted downstream. Otherwise it's dropped.
5. **Format & Output**: Emitted events are stripped of internal fields, serialized back to JSON, and written to `output.jsonl`.

**Dedup logic** (`event_timestamp <= previous_timestamp + 10s` → drop):
- Same `event_id` within ≤10s → dropped
- Same `event_id` after >10s → emitted (window restarts)
- Different `user_id` → independent state
