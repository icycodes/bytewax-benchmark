## Implementation Summary

**File:** `rate_limiter.py`

**Architecture:**
1. Reads JSONL input via `TestingSource` over file lines
2. Parses each line as JSON with `op.map`
3. Keys the stream by `user_id` (as string) using `op.key_on`
4. Applies the Token Bucket algorithm via `op.stateful_map` with per-key state `(current_tokens, last_timestamp)`
5. Drops the key and serializes back to JSON
6. Writes output via a custom `FileSink` (implements `DynamicSink` + `StatelessSinkPartition`)

**Token Bucket Logic:**
- Initial state: 10.0 tokens at the first event's timestamp
- On each event: refill = `(timestamp - last_timestamp) * 2.0`, capped at 10.0
- If `cost == -1.0`: reset bucket to full, always `allowed: true`
- If `tokens >= cost`: deduct cost, `allowed: true`
- Otherwise: no deduction, `allowed: false`
- Negative elapsed time (clock skew) is clamped to 0

**Usage:**
```
python rate_limiter.py --input <input.jsonl> --output <output.jsonl>
```
