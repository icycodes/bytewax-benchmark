**Key design decisions:**

1. **`op.join` with `emit_mode="complete"`** — Only emits a combined record when *both* an impression and a click have been observed for a given `user_id`. This implements the required inner join semantics.

2. **`op.map_value`** (not `op.map`) — Keeps the stream keyed as `(key, value)` tuples, which is required by `op.output` for routing. The value is transformed into a JSON string containing `user_id`, `impression`, and `click`.

3. **`FileSource` / `FileSink`** — Bytewax's built-in file connectors handle line-by-line I/O with proper partitioning.

4. **Default `insert_mode="last"`** — When multiple events arrive for the same key on one side, only the latest value is retained for the join.
